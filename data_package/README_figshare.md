## Description

This dataset is a structured inventory of Australian birds, including all species and subspecies (ultrataxa) with ecological or conservation relevance within Australian jurisdictions. For each taxon, the dataset provides stable unique identifiers, scientific and common names, higher-order taxonomic classifications, spatial bird group and population descriptors, coastal range classification, an external Avibase identifier, and Australian Red List Index (RLI) extinction risk values across four assessment periods (1990, 2000, 2010, and current).

The list follows the taxonomy of [AviList](https://avilist.org) (AviList Core Team 2025) as its primary global reference for species-level classification, with a small number of transparent, documented deviations where adoption would compromise conservation relevance, data integrity, or biological resolution (see `avilist_changes.csv` and `wlist_avilist_changes.pdf`).

### Purpose

The primary purpose of this list is to provide a stable and consistent taxonomic framework for organising biodiversity data while accommodating ongoing changes in taxonomy. Taxonomy is treated as an evolving scientific framework, whereas ecological observations and conservation data must remain interpretable through time. Accordingly, this list separates persistent biological observations from changing taxonomic interpretations, using:

- stable, unique identifiers for all taxa (`taxon_id`),
- taxa recorded at the finest practical resolution (ultrataxa), and
- alignment with global taxonomies with transparent, limited deviations.

This approach supports long-term data usability, reproducibility of analyses, and integration across spatial range layers, ecological trait databases, conservation assessments, and bird occurrence records.

### Biological units: the ultrataxon concept

A key feature of this dataset is its use of **ultrataxa** — terminal taxonomic units defined as either a monotypic species (a species with no subspecies) or a subspecies. Both are treated as equally valid for data management and conservation purposes, because conservation value is not determined by taxonomic rank. In Australia, legislation recognises taxa at species, subspecies, and population levels; many of Australia's most imperilled birds are subspecific.

The `is_ultrataxon` flag identifies these terminal units. Non-ultrataxon rows are species-level rows that serve as the parent grouping entry for polytypic species (i.e. species with recognised subspecies). The `taxon_id` encodes this structure: ultrataxon identifiers are prefixed with `u`; the numeric component corresponds to `sp_id`.

## Methods summary

The dataset was compiled through systematic synthesis of published global and regional taxonomies, authoritative subspecies inventories, and successive Australian bird conservation assessments. Species-level classifications follow AviList (AviList Core Team 2025), itself drawn from the HBW/BirdLife Illustrated Checklist. Subspecies listings draw on Schodde and Mason (1997, 1999), the Handbook of Australian, New Zealand and Antarctic Birds, and published taxonomic works. Extinction risk assessments were sourced from successive Action Plans for Australian Birds (Garnett 1992; Garnett and Crowley 2000; Garnett et al. 2011; Garnett and Baker 2021), with interim updates applied. All taxa are assigned stable unique identifiers; taxonomic names and classifications are systematically vetted against global lists with reconciliation via Avibase. AviList v1 integration changes are documented in `avilist_changes.csv`.

The operational database is a PostgreSQL instance. Package outputs (CSVs and SQL dump) are generated by `package_wlist.py`.

## How to use this dataset

### General users (CSV)

The primary data file is `wlist_core.csv`. Open in any spreadsheet application, R, Python, or GIS software.

To decode the `austRli`, `austRli1990`, `austRli2000`, and `austRli2010` fields, join on `lut_rli.csv` using `austRli = id`.

```python
import pandas as pd

wlist = pd.read_csv("wlist_core.csv")
rli   = pd.read_csv("lut_rli.csv")

# austRliStatusDesc is already included in wlist_core.csv as a convenience column
# To re-derive or join for a specific period:
wlist_with_rli = wlist.merge(rli, left_on="austRli1990", right_on="id", how="left", suffixes=("", "_1990"))
```

### PostgreSQL users

To recreate the full database with all constraints, triggers, and procedures:

```bash
# 1. Create schema only (recommended first step)
psql -U your_user -d your_db -f wlist.ddl

# 2. Import data (schema + INSERT rows)
psql -U your_user -d your_db -f wlist_core.sql
```

`wlist.ddl` includes the complete constraint set; `wlist_core.sql` is a standalone dump with a simpler schema. For the full operational schema (including triggers and stored procedures), use `wlist.ddl`.

### Adding taxa with `add_taxon`

Once the database is restored from `wlist.ddl` (or `wlist_core.sql`), new rows can be inserted using the Python utility in `add_taxon/`. It wraps the `wlist_add_row` stored procedure with batch processing, all-or-nothing transactions, pre-commit validation, and a dry-run mode.

```bash
# install dependencies
pip install -r requirements.txt

# set database connection (replace placeholders)
export DCORE_HOST=localhost
export DCORE_DB=wlist
export DCORE_USER=<your_db_user>
export DCORE_PASSWORD=<your_db_password>

# validate without committing
python add_taxon/add_taxon.py --csv add_taxon/sample_add_taxon.csv --dry-run

# commit
python add_taxon/add_taxon.py --csv your_rows.csv
```

See `add_taxon/README.md` for the full field reference, JSON input format, and the Python API. Row deletion is handled by the `wlist_delete_row` stored procedure directly — no Python wrapper is currently provided.

### Filtering to core taxa only

The `population` field encodes the ecological/endemism status of each taxon. To restrict analysis to extant core Australian taxa, filter out rows where `population` contains `Extinct`, `Failed introduction`, or `Vagrant`:

```python
core = wlist[~wlist["population"].str.contains(
    "Extinct|Failed introduction|Vagrant", na=False
)]
```

### Understanding `taxon_id` and `sp_id`

- `sp_id` — integer unique to each species (same value shared across a species and all its subspecies).
- `taxon_id` — unique identifier for each row. Format: `[u]<sp_id>[a|b|c...]`
  - `u` prefix = ultrataxon (terminal unit: monotypic species or subspecies)
  - No `u` prefix = polytypic species-level parent row
  - Alphabetical suffix (`a`, `b`, `c`…) distinguishes subspecies of the same species

## Field reference

### wlist_core.csv (20 fields)

CSV field names use camelCase. Database equivalents use snake_case (e.g. `taxonSort` ↔ `taxon_sort`).

| Field | Type | Description |
|-------|------|-------------|
| `taxonSort` | integer | Taxonomic sort order |
| `isUltrataxon` | smallint | 1 = terminal unit (ultrataxon); NULL = polytypic species parent |
| `taxonLevel` | varchar | `sp` = species; `ssp` = subspecies |
| `spId` | smallint | Unique species identifier (shared across species and its subspecies) |
| `taxonId` | varchar | Unique taxon identifier — primary key |
| `taxonName` | varchar | Common name |
| `taxonScientificName` | varchar | Scientific name |
| `familyName` | varchar | Family common name |
| `familyScientificName` | varchar | Family scientific name |
| `order` | varchar | Taxonomic order |
| `avibaseId` | varchar | External [Avibase](https://avibase.bsc-eoc.org) identifier |
| `notes` | text | General notes |
| `birdGroup` | varchar | Spatial bird group — see below |
| `population` | varchar | Population/endemism qualifier — see below |
| `isCoastal` | varchar | Coastal distribution descriptor — see below |
| `austRli1990` | smallint | Australian Red List Index code, 1990 assessment |
| `austRli2000` | smallint | Australian Red List Index code, 2000 assessment |
| `austRli2010` | smallint | Australian Red List Index code, 2010 assessment |
| `austRli` | smallint | Australian Red List Index code, current assessment |
| `austRliStatusDesc` | varchar | Human-readable RLI category (joined from `lut_rli`) |

#### `birdGroup` values

| Value | Description |
|-------|-------------|
| Terrestrial | Taxa with polygonal distributions dependent on terrestrial habitats (Garnett et al. 2015) |
| Wetland | Taxa with polygonal distributions dependent on inland water habitats (Garnett et al. 2015) |
| Shoreline (migratory/resident) | Taxa associated with linear coastal or inland water shoreline habitats |
| Marine | Taxa feeding primarily in marine habitats (Garnett et al. 2015) |

#### `population` values

| Value | Description |
|-------|-------------|
| Australian | Australian breeding population of taxa that also breed elsewhere |
| Endemic | Taxa occurring in Australia and nowhere else |
| Endemic (breeding only) | Taxa breeding only in Australia but frequently dispersing elsewhere |
| Extinct (endemic) | Naturally occurring Australian endemics now globally extinct |
| Extinct (Australia only) | Naturally occurring Australian taxa now extinct in Australia but persisting elsewhere |
| Failed introduction | Introduced taxa for which wild populations have not persisted |
| Introduced | Taxa introduced since 1788 with established self-sustaining wild populations |
| Non-breeding | Taxa occurring regularly in Australia but breeding elsewhere |
| Vagrant | Non-breeding taxa crossing into Australian jurisdictions in insignificant numbers (<0.1% global population) |
| Vagrant (from NZ introduction) | Vagrants derived from populations introduced to New Zealand |

#### `isCoastal` values

| Value | Description |
|-------|-------------|
| Obligate | Strictly coastal; never found away from coasts |
| Facultative | Principally coastal but less frequently found inland |
| *(null)* | Not applicable |

### lut_rli.csv (3 fields)

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | RLI code (matches `austRli*` fields in `wlist_core.csv`) |
| `code` | text | IUCN two-letter code |

| id | code | category |
|----|------|----------|
| 0 | LC | Least Concern |
| 1 | NT | Near Threatened |
| 2 | VU | Vulnerable |
| 3 | EN | Endangered |
